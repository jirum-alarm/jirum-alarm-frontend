## 작업내용

- git status와 git diff로 변경사항 파악
- git log로 기존 커밋 메세지 스타일 확인

## 커밋 컨벤션

- 컨벤셔널 커밋 형식 적용:
  - 타입: chore, feat, fix, style, refactor 등
  - 범위: 해당되는 경우 (선택적)
  - 설명: 변경사항의 핵심 요약
- 적절한 커밋 메세지 작성 후 커밋 메세지만 추천

## 유의 사항

- 절대 커밋을 하지 않습니다. 커밋 메세지 추천만 해줍니다.
